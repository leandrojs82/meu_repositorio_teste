Cheat Sheet

Guia Rápido — Markdown (GFM)

Sintaxe principal do Markdown com extensões do GitHub Flavored Markdown. Copie os blocos e use no seu README ou Wiki.

Títulos Ênfase Listas Links & Imagens Código (inline e bloco) Citações (blockquote) Tabelas Linhas horizontais Listas de tarefas Notas de rodapé HTML embutido Extras do GitHub Diagramas Mermaid Escapar caracteres

Títulos

Use # (1 a 6) antes do texto.

# Título H1
## Título H2
### Título H3
#### Título H4
##### Título H5
###### Título H6

Ênfase

Texto em *itálico* ou _itálico_
Texto em **negrito** ou __negrito__
Texto com ***negrito e itálico***
Texto ~~riscado~~

Dica: use o estilo consistente no seu projeto.

Listas

Não ordenadas

- Item A
- Item B
  - Subitem B1
  - Subitem B2
* Marcadores também podem usar asterisco
+ Ou sinal de mais

Ordenadas

1. Primeiro
2. Segundo
3. Terceiro
   1. Subitem 3.1
   2. Subitem 3.2

Quebras de linha

Use dois espaços no fim da linha ou um <br>.

Linha com quebra··
nova linha

Outra forma:<br>
continuação

Links & Imagens

[Texto do link](https://exemplo.com)
[Título opcional](https://exemplo.com "Título ao passar o mouse")

![Alt da imagem](https://exemplo.com/imagem.png)
![Alt com título](https://exemplo.com/img.png "Descrição")

Use um texto alternativo (alt) descritivo para acessibilidade.

Código

Inline

Use o acento grave: `console.log("Olá")`

Blocos com linguagem

```bash
git status
git add .
git commit -m "mensagem"
```
```javascript
function somar(a, b) {
  return a + b;
}
```

Em muitos renderizadores, indicar a linguagem ativa syntax highlight.

Citações (blockquote)

> Uma citação simples.
> 
> - Pode ter múltiplas linhas
> - E até listas
> 
> > Citação aninhada

Tabelas (GFM)

| Coluna A | Coluna B | Coluna C |
|----------|:--------:|---------:|
| esquerda | centro   | direita  |
| texto    | **bold** | 123      |

: define alinhamento (esquerda, centro, direita).

Diagramas Mermaid (GitHub)

```mermaid
flowchart LR
  A[Início] --> B{Decisão?}
  B -->|Sim| C[Passo 1]
  B -->|Não| D[Passo 2]
```

Em repositórios GitHub, Mermaid é renderizado nativamente.

Escapar caracteres

Anteponha \ para exibir caracteres literais.

\*não em itálico\* \# não vira título
\`código literal\`

Como usar este arquivo

Salve como markdown-cheatsheet.html na raiz do repositório (ou em docs/) e abra no navegador.

Dica: adicione um link no README.md para acesso rápido.

Mantenha um padrão de estilo (títulos, listas e código) para consistência entre repositórios.
