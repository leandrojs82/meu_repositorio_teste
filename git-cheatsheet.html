Cheat Sheet

Guia Rápido de Git — Comandos Essenciais

HTML pronto para colocar no repositório. Use os blocos abaixo para copiar os comandos mais usados no dia a dia.

Início rápido

Configure sua identidade e crie/clone um repositório.

git config --global user.name "Seu Nome"
git config --global user.email "voce@exemplo.com"
git config --global init.defaultBranch main

# criar repositório local
git init

# clonar um repositório existente
git clone https://github.com/usuario/projeto.git

Ciclo básico

Fluxo de trabalho padrão: verificar, adicionar, commitar e sincronizar.

# status do repositório
git status

# adicionar alterações (pontual/geral)
git add arquivo.txt
git add .

# criar commit
git commit -m "mensagem do commit"

# obter atualizações remotas (sem mesclar)
git fetch

# integrar/atualizar a branch local
git pull        # (por padrão faz merge)
git pull --rebase

# enviar commits para o remoto
git push origin main

Branches & Merge

# listar branches locais e remotas
git branch
git branch -r

# criar e trocar de branch
git switch -c minha-branch        # (ou: git checkout -b minha-branch)
git switch main                   # voltar para main

# enviar branch para o remoto
git push -u origin minha-branch

# mesclar outra branch na atual
git merge minha-branch

# excluir branch já mesclada
git branch -d minha-branch
git push origin --delete minha-branch

Remoto

# verificar remotos
git remote -v

# adicionar/alterar remoto
git remote add origin https://github.com/usuario/projeto.git
git remote set-url origin https://github.com/usuario/novo.git

# sincronizar referências
git fetch --all --prune

# atualizar branch local rebaseando mudanças remotas
git pull --rebase origin main

# primeiro push (cria upstream)
git push -u origin main

Stash

# salvar alterações temporariamente
git stash
git stash push -m "WIP: ajuste login" --include-untracked

# listar e aplicar
git stash list
git stash show -p
git stash apply
git stash pop

# remover
git stash drop
git stash clear

Log & Diff

# histórico resumido e visual
git log --oneline --graph --decorate --all

# detalhes de um commit/arquivo
git show <hash>
git blame caminho/arquivo.ext

# diferenças
git diff                   # não indexado
git diff --staged          # pronto para commit

Desfazer (restore/reset/revert)

# descartar mudanças locais em um arquivo
git restore caminho/arquivo.ext

# tirar arquivo da área de stage
git restore --staged caminho/arquivo.ext

# voltar N commits mantendo alterações como "não stageadas"
git reset --soft HEAD~1

# voltar N commits descartando alterações
git reset --hard HEAD~1

# desfazer um commit específico através de um commit inverso
git revert <hash>

Atenção: reset --hard remove alterações locais sem volta. Use com cuidado.

Tags & Releases

# criar e listar tags
git tag -a v1.0.0 -m "primeiro release"
git tag

# enviar tags
git push origin v1.0.0
git push origin --tags

Rebase

# atualizar sua branch com a main
git fetch origin
git rebase origin/main

# rebase interativo (editar/combinar commits)
git rebase -i HEAD~5

Cherry‑pick

# aplicar um commit específico na branch atual
git cherry-pick <hash>

Submódulos

# adicionar submódulo
git submodule add https://github.com/usuario/lib.git libs/lib

# clonar/atualizar submódulos
git submodule update --init --recursive

.gitignore

# exemplos de padrões
node_modules/
dist/
*.log
.env

# se já foi versionado, remova do índice e commite novamente
git rm -r --cached .
git add .
git commit -m "Respeitar .gitignore"

Config & credenciais

# ver configurações
git config --list

# normalização de fim de linha (Windows)
git config --global core.autocrlf true

# Linux/macOS (sugestão)
# git config --global core.autocrlf input

# cache/gerenciador de credenciais (Windows)
git config --global credential.helper manager-core

SSH (opcional)

# gerar chave
ssh-keygen -t ed25519 -C "seu-email@exemplo.com"

# iniciar o agente e adicionar a chave
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

Problemas comuns & Dicas

  • Pull criando merge commit indesejado: use git pull --rebase.
  • “Permission denied (publickey)”: configure SSH ou use HTTPS com o gerenciador de credenciais.
  • Conflitos de merge: edite arquivos marcados, git add e finalize com git merge --continue ou git rebase --continue.
  • Apontar a branch principal para main: git symbolic-ref refs/remotes/origin/HEAD refs/remotes/origin/main (quando necessário).
  • Trabalhe em branches pequenas: facilita code review e evita conflitos grandes.
