Guia Rápido — GitHub Desktop

Fluxos essenciais no app gráfico do GitHub. Ideal para colocar no seu repositório e compartilhar com o time.

Instalação & login

  1. Baixe em desktop.github.com e instale.
  2. Abra o aplicativo e faça login com sua conta do GitHub.
  3. Opcional: conecte contas corporativas/Enterprise se necessário.
O GitHub Desktop funciona como uma camada visual para os comandos Git mais usados.

Conceitos & interface

  • Current Repository: repositório selecionado (canto superior esquerdo).
  • Current Branch: branch ativa.
  • Changes: arquivos modificados e mensagem de commit.
  • History: histórico de commits e detalhes.
  • Fetch/Push/Pull: botões no topo sincronizam com o remoto.

Criar, Clonar ou Abrir

  1. Clone: File → Clone repository… (URL/Enterprise/GitHub.com).
  2. Open um repo existente: File → Add local repository….
  3. Create um novo repo: File → New repository….

CLI equivalente: git clone <url> · git init

Ciclo básico (commit → push → pull)

  1. Na aba Changes, selecione os arquivos que deseja comitar (checkbox).
  2. Escreva uma mensagem de commit clara.
  3. Clique em Commit to <sua-branch>.
  4. Push para enviar ao remoto (botão no topo).
  5. Pull/Fetch para trazer atualizações do remoto.

CLI equivalente: git add . · git commit -m "..." · git push · git pull

Branches

  1. Clique no seletor de branch (topo) e New Branch.
  2. Dê um nome (ex.: feature/login) e crie.
  3. Para publicar no remoto, clique em Publish branch.
  4. Para trocar de branch, use o seletor e escolha a branch desejada.
  5. Para excluir: Branch → Delete… (após merge ou quando seguro).

CLI equivalente: git switch -c feature/login · git push -u origin feature/login

Pull Requests

  1. Com sua branch publicada, clique em Create Pull Request no topo (ou Branch → Create Pull Request…).
  2. O app abrirá o navegador em GitHub.com para você preencher título, descrição e enviar o PR.
  3. Após o merge no GitHub, volte ao Desktop e Pull na main para trazer as mudanças.

CLI equivalente: abrir PR via GitHub CLI (gh pr create) ou no site.

Resolver conflitos

  1. Ao fazer Pull/Merge com conflito, o Desktop lista os arquivos conflitantes.
  2. Escolha Open in editor para resolver manualmente ou use as opções de escolher um lado.
  3. Marque o arquivo como resolvido (checkbox) e faça o Commit merge.

CLI equivalente: editar marcadores <<<<<<<, git add, git merge --continue ou git rebase --continue.

Descartar/Restaurar mudanças

  • Descartar arquivo: clique direito no arquivo em Changes → Discard Changes.
  • Descartar tudo: menu Repository → Discard All Changes… (cuidado!).
  • Restaurar um arquivo parcialmente: selecione linhas no diff e clique em Discard Selected Lines ou Stage Selected Lines.

CLI equivalente: git restore <arquivo> · git checkout -- <arquivo> (legado)

Stash ao trocar de branch

Ao trocar de branch com mudanças locais, o Desktop oferece a opção “Stash changes to switch branches”. Marque para guardar temporariamente.

  • Depois, para aplicar: o app sugere Pop stash ao voltar.

CLI equivalente: git stash · git stash pop

Histórico & Reverter commit

  • Aba History para visualizar commits e arquivos alterados.
  • Reverter commit: clique direito em um commit → Revert This Commit (o app cria um novo commit inverso).
  • Copiar hash: clique direito no commit → Copy SHA.

CLI equivalente: git log --oneline · git show <sha> · git revert <sha>

Remotos & publicar

  • Publish repository para criar o remoto no GitHub.
  • Repository → Repository Settings… → Remote: ver/alterar URL, adicionar outros remotos.
  • Fetch origin sincroniza referências sem mesclar.
  • Pull traz e integra mudanças da branch remota.
  • Push envia seus commits.

CLI: git remote -v · git remote set-url origin <url> · git fetch · git pull · git push

Abrir no editor

  • Open in Visual Studio Code (ou editor configurado) pelo botão na barra superior.
  • Configure o editor padrão: File → Options… → Integrations → External Editor.
  • Abra no Explorer/Finder: Repository → Show in Explorer.

Configurações úteis

  • Options → Accounts: gerenciar logins GitHub/Enterprise.
  • Options → Integrations: editor e shell padrão (Git Bash, PowerShell, etc.).
  • Options → Appearance: tema claro/escuro e preferências.
  • Repository Settings: Default branch, Ignored files (abre o .gitignore no editor), remotos.

Equivalentes no Git CLI (mapa rápido)

Criação/Clone

Desktop: New repository / Clone repository
CLI:     git init
         git clone <url>

Commit & Sync

Desktop: Changes → Commit → Push/Pull/Fetch
CLI:     git add .
         git commit -m "mensagem"
         git push / git pull / git fetch

Branches

Desktop: New Branch / Publish Branch / Switch Branch
CLI:     git switch -c minha-branch
         git push -u origin minha-branch
         git switch outra-branch

Reverter

Desktop: History → Right click → Revert This Commit
CLI:     git revert <sha>

Conflitos

Desktop: Resolve conflicts in editor → Commit merge
CLI:     editar conflitos → git add → git merge --continue

Problemas comuns & dicas

  • Sem botão Push: não há commits locais para enviar.
  • Branch “ahead/behind”: use Fetch/Pull para alinhar com o remoto.
  • “Publish branch” aparece: a branch ainda não existe no remoto; publique.
  • Conflitos frequentes: faça Pull antes de começar e crie branches menores.
  • .gitignore não funcionou: arquivo já versionado — remova do índice e commite novamente (veja o cheat de Git CLI).
